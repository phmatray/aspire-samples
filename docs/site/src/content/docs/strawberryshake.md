The Infrastructure layer talks to Strapi's GraphQL API through a **typed client
generated by StrawberryShake 16**. The package reference in
`AspireStrapi.Infrastructure.csproj` is:

```xml
<PackageReference Include="StrawberryShake.Blazor" Version="16.0.11" />
```

StrawberryShake 16 uses **net10-native code generation** — codegen runs as part
of the normal `dotnet build` against `net10.0`, with no separate global tool or
MSBuild task version-skew to manage.

## Configuration

Codegen is driven by `dotnet/AspireStrapi.Infrastructure/ApiClient/.graphqlrc.json`:

```json
{
  "schema": "schema.graphql",
  "documents": "**/*.graphql",
  "extensions": {
    "strawberryShake": {
      "name": "BlogClient",
      "namespace": "AspireStrapi.Infrastructure.ApiClient",
      "url": "http://localhost:1337/graphql/",
      "records": { "inputs": false, "entities": false },
      "transportProfiles": [
        { "default": "Http", "subscription": "WebSocket" }
      ]
    }
  }
}
```

- `schema.graphql` is the introspected Strapi 5 schema (flattened).
- Every `*.graphql` document in the folder becomes a strongly typed operation.
- The generated client is named **`BlogClient`**.

## The typed queries

The `.graphql` documents (e.g. `query-articles.graphql`) define operations such
as `GetArticles`, `GetArticlesByCategory`, and `GetArticleById`. Because the
Strapi 5 schema is flattened, the generated result types expose flat fields:

```csharp
var result = await _blogClient.GetArticles.ExecuteAsync(ct);
foreach (var article in result.Data!.Articles)
{
    // article.Title, article.Slug, article.PublishedAt
    // article.Cover?.Url, article.Category?.Name, article.Author?.Name
}
```

There is no `.Data.Attributes` indirection — the v4 wrapping is gone.

## The raw body query

The article **body** is a Strapi **dynamic zone** (`blocks`), a GraphQL union of
component types. StrawberryShake's generated mapper mishandles this union, so
the body is fetched with a small **hand-written** GraphQL request in
`StrapiArticleBodyClient`:

```graphql
query ArticleBlocks($documentId: ID!) {
  articles(filters: { documentId: { eq: $documentId } }) {
    blocks {
      __typename
      ... on ComponentSharedRichText { body }
      ... on ComponentSharedQuote { title body }
      ... on ComponentSharedMedia { file { url alternativeText } }
    }
  }
}
```

The adapter posts this with `System.Text.Json` using **camelCase serialization**
so the request body is `{"query": ..., "variables": ...}` — GraphQL requires
those lowercase keys. The result is rendered to HTML behind the article
repository port, so the union never leaks into the Application or Domain layers.
